# nnUNet-C rebuild runbook (iso-plan grid, from build-dataset)

Full, ordered rebuild after the CNISP native-mapping fix + the train-grid revert
(train now assembles on the **835 iso plan** grid, matching test). No step is
skipped; stale artifacts are deleted first; the CNISP input is regenerated
**from saved latents** (no re-optimization) since only the mapping changed.

Do control **C** (Dataset845) and **B** (Dataset855). C is shown in full; B notes
follow each step.

Prereqs (GPU/data host):
```bash
export nnUNet_raw=/fs5/p_masi/linz18/EyeSegmentation/nnUNet_raw
export nnUNet_preprocessed=/fs5/p_masi/linz18/EyeSegmentation/nnUNet_preprocessed
export nnUNet_results=/fs5/p_masi/linz18/EyeSegmentation/nnUNet_results
```

## Key facts (why this runbook)
- The network plan is the **835 iso plan** (`nnUNetPlans_iso05`, spacing
  **0.4765625** iso), pulled in by `build_finetune_plan.py` via
  `corrector.yaml::reference_plan_json: nnUNetPlans_iso05`. Confirm that key is set.
- The 5ch assembly spacing (train + test + CNISP iso emit) now defaults to that
  plan spacing via `resolve_target_spacing` — **not** a hard-coded 0.5. Don't pass
  `--iso-mm 0.5` / `ISO_MM=0.5` anywhere; let it resolve from the plan.
- Only the **mapping** changed, so CNISP masks are regenerated by **re-decoding +
  re-mapping the SAVED latents** (`REMAP_FROM_LATENT=1` for train / 032,
  `RESUME_FROM_LATENT=1` for test / 03_infer). No latent re-optimization, no
  re-inference. (If you also changed the CNISP model/training, the latents are
  stale → do a full re-infer instead.)
- Train ch1..4 now come from CNISP's **iso-DIRECT decode** (the same decode path,
  incl. observed-meta reframe, that produces the TEST ch1..4), NOT the old
  "native decode → resample to iso". This removes the train↔test decode-path
  drift and the redundant resample. Generated by 032's `--emit-iso-prelabel-dir`
  (auto via `run_corrector_cnisp.sh EMIT_ISO=1`), consumed by
  `build_corrector_dataset --prelabel-grid iso` (default).

## Step 0 — confirm config
```bash
grep -E "reference_plan_json|reference_plan|target_spacing" nnunet-c/configs/corrector.yaml
# expect: reference_plan_json: nnUNetPlans_iso05   (and target_spacing: null)
```

## Step 1 — delete stale artifacts (so nnUNet cannot reuse the old GT-grid build)
Only the 845/855 dataset dirs; nothing else is touched. Back up old results if you
want them for comparison.
```bash
# control C (845)
rm -rf "$nnUNet_raw/Dataset845_PHOTON_CT_CORR_C_cnisp"
rm -rf "$nnUNet_preprocessed/Dataset845_PHOTON_CT_CORR_C_cnisp"
mv "$nnUNet_results/Dataset845_PHOTON_CT_CORR_C_cnisp" \
   "$nnUNet_results/Dataset845_PHOTON_CT_CORR_C_cnisp__old_gtgrid" 2>/dev/null || true
```
B: same with `Dataset855_PHOTON_CT_CORR_B_stacked`.

## Step 2 — regenerate the CNISP input FROM SAVED LATENTS (mapping-only, no re-infer)
Control **C** ch1–4 come from CNISP. Re-decode + re-map the saved latents and
OVERWRITE the buggy masks (reuses `<out>/latent/`; a (source,step,eye) without a
saved latent is skipped):
```bash
REMAP_FROM_LATENT=1 bash nnunet-c/run_corrector_cnisp.sh
# From the SAVED latents (no re-optimization), re-run mapping and write BOTH:
#   data/cnisp_pred/*.nii.gz                     native masks (nnUNet {1,2,3,4})
#   data/cnisp_pred_train_iso/native_space_step_XX/  iso-DIRECT prelabels
#     (+ manifest_by_source/<sid>.json)          EMIT_ISO=1 by default
```
- No separate scheme remap: 032 saves `data/cnisp_pred` already remapped BY NAME
  (canonical {1:ON,2:Globe,3:Fat,4:Recti} -> nnUNet {1:ON,2:Recti,3:Globe,4:Fat}).
- The **iso** prelabels are what train ch1..4 now use (Step 3, `--prelabel-grid
  iso`), generated by the SAME iso-decode path (incl. observed-meta reframe) that
  `infer.py` uses for the TEST prelabels -> train/test ch1..4 match, no native->iso
  resample. `EMIT_ISO=0` to skip; `ISO_MM` defaults to the 835 iso plan spacing.

B: control B's ch1–4 = nnUNet prediction (`data/nnunet_pred`, from
`run_corrector_data.sh predict`) — **no CNISP, nothing to remap here**. Just make
sure `data/nnunet_pred` exists.

> Full re-infer (ONLY if latents are stale, e.g. CNISP model changed):
> `SKIP_EXISTING=0 bash nnunet-c/run_corrector_cnisp.sh`  (slow: re-optimizes latents)

## Step 3 — rebuild the TRAIN dataset (iso-plan grid + iso-DIRECT ch1..4)
```bash
python nnunet-c/scripts/build_corrector_dataset.py --control C
# --prelabel-grid iso is the DEFAULT: ch1..4 = CNISP iso-direct decode
# (data/cnisp_pred_train_iso, remapped by name) -- identical decode path to test.
# log: grid = iso [0.4765625,...] + prelabel = CNISP iso-direct decode
# (legacy native mask resampled: add --prelabel-grid native)
```
B:
```bash
python nnunet-c/scripts/build_corrector_dataset.py --control B --require-cnisp
```

## Step 4 — train all folds (fingerprint → plan_experiment → build_finetune_plan
(from iso05) → preprocess → gate → surgery → train; nothing skipped)
```bash
bash nnunet-c/run_train.sh C 0
for f in 1 2 3 4; do SKIP_PREPROCESS=1 bash nnunet-c/run_train.sh C $f; done
```
B: same with `B`.

## Step 5 — predict + eval (test iso prelabels re-mapped from latents; current best ckpt)
```bash
RESUME_FROM_LATENT=1 bash nnunet-c/run_corrector_predict.sh C 0
# prints "iso spacing (emit + testset assembly) = 0.4765625 mm" and the checkpoint path
```
B (no CNISP step):
```bash
bash nnunet-c/run_corrector_predict.sh B 0
```

## Guardrails
- **train ⟷ test iso spacing must match** — both now auto-resolve to the 835 iso
  plan (0.4765625). Do not override one side with `--iso-mm` / `ISO_MM` unless you
  set the same value on the other.
- **remap-from-latent** reuses saved latents (fast, mapping-only). It is valid
  ONLY if the latents were optimized with the current CNISP model. If the CNISP
  model/training changed, re-infer instead.
- Step 1 `rm -rf` touches ONLY the 845/855 dataset dirs (regenerated below); it
  does not touch Dataset835, CNISP outputs, saved latents, or other datasets.
- The `__old_gtgrid` backup lets you A/B the old GT-grid model vs the new iso one.
